# Regression coefficient matrix β (z-scored, ridge α = 1000)

OLS-with-ridge fit of `y = β x` where:
- `x = (z_24, 1)` — 24 z-scored ask prices at t = −10 min plus a bias term.
- `y` = 24-vector of binary outcomes (`V24[:, world_idx]`).
- Dataset: telonex `train` split, fully-stale rows dropped, `n = 2107`.
- **Penalty**: `α‖β_features‖² = 1000 · ‖β_features‖²`. Bias unregularised.

β is expressed in the basis:
- **rows** = canonical token basis (12 YES + 12 NO outcome positions)
- **columns** = covariance-eigenvector basis of z-scored asks (PC1..PC24)
  + final `bias` column for the intercept.

Run with `python -m strategies.pregame_dc.regression --ridge 1000`.
Source CSV: `strategies/pregame_dc/beta_token_x_pc_ridge1000.csv`.
Heatmap: `strategies/pregame_dc/plots/beta_token_x_pc_ridge1000.png`.

## Why α = 1000

In the PC basis, ridge multiplies each PC's OLS coefficient by
`λ_k(F^T F) / (λ_k(F^T F) + α)`, where `λ_k(F^T F) ≈ (n − 1) · λ_k(cov(Z))`.

| PC | cov-eig λ_k | retain factor | coefficient kept |
|----|------------:|--------------:|:---|
| PC1 (tilt) | 6.46 | 13600 / 14600 | **0.93×** — barely shrunk |
| PC2 (book width) | 5.24 | 11026 / 12026 | 0.92× |
| PC3 (totals) | 3.85 | 8118 / 9118 | 0.89× |
| PC4 | 1.05 | 2215 / 3215 | 0.69× |
| PC5 | 0.95 | 1990 / 2990 | 0.67× |
| PC10 | 0.57 | 1192 / 2192 | 0.54× |
| PC15 | 0.26 | 549 / 1549 | 0.35× |
| PC20 | 0.07 | 154 / 1154 | 0.13× |
| PC24 | 0.02 | 46 / 1046 | **0.04×** — almost zeroed |

**Effective DoF: 10.76 / 24.** The "real" 3-factor backbone is preserved,
the noise-floor tail is suppressed, and the dimensionality drop matches
the rank-≈10 spectrum we already saw.

## What changed vs OLS

(see `beta_matrix.md` for the OLS standardized version)

- **Top-3 PCs barely move.** PC1 A-win-Y: −0.063 (OLS) → −0.059 (ridge). PC3 O 2.5 Y: +0.046 → +0.041. The 7–11% shrinkage on the leading directions just nudges them.
- **Tail PCs collapse.** Where OLS has e.g. PC22 Draw-Y at +0.058 and PC24 A-2.5-Y at −0.083, ridge gives +0.006 and −0.004 respectively. Two orders of magnitude smaller. This kills the OLS tail-PC noise amplification.
- **Bias column unchanged.** Intercept = empirical base rate, unregularised by construction.
- **Y / N rows are still exact negatives.** Mechanical from `Y_NO = 1 − Y_YES`; unaffected by ridge.

## Summary

Ridge α = 1000 gives the same story OLS told but cleaner: PC1 (tilt) and PC3 (totals) carry the moneyline / spread / over-under predictive coefficients, PC2 (book width) is non-directional, and PC4+ are essentially silenced. This is the basis I'd actually use for downstream analysis or as a baseline for a structured model.
